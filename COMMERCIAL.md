# Pisces Moon OS — Commercial Licensing

Pisces Moon OS is open-source under **AGPL-3.0-or-later**. Most users — individuals, hobbyists, hackerspaces, research groups, government agencies, schools, nonprofits, and open-source projects — can use it without ever talking to us. That's intentional. The protocol is local-first, no gatekeepers.

But AGPL-3.0 has obligations that don't fit every situation. **If you're a commercial entity and any of the following apply, you probably need a commercial license:**

- You want to integrate Pisces Moon OS into a closed-source product
- You want to ship Pisces Moon OS (or a fork) to customers without releasing your modifications under AGPL
- You want to offer Pisces Moon OS as part of a managed service (SaaS, MSP, hosted offering) without making your service-side modifications public
- You need a different license for legal/audit reasons (some enterprises forbid AGPL across the board)
- You need indemnification, warranty, or formal support agreements

A commercial license removes these obligations and adds:

- **Right to use under terms compatible with proprietary or closed-source products**
- **Indemnification** for the licensed code
- **Priority support** (SLA depends on tier)
- **Custom feature development** at agreed rates
- **Branding rights** for OEM / white-label integrations
- **Direct line to the maintainer** for architecture review and security questions

---

## What's NOT a commercial use

You do **not** need a commercial license if you're:

- Running Pisces Moon OS on your own devices for your own use
- Forking it on GitHub and distributing under AGPL-3.0
- Modifying it and using the modified version privately
- Using it inside your organization for internal tools (as long as you're not running it as a service for outside users)
- Building open-source apps on top of the HTML suite
- Teaching, researching, or writing about it
- Running a one-off field deployment for an event, expedition, or community group
- Using the bundled third-party libraries (Leaflet, ZXing, etc.) under their own licenses, separately from Pisces Moon OS

When in doubt, **AGPL-3.0 is fine for the vast majority of users.** This commercial path exists for the cases where it explicitly isn't.

## Why AGPL + Commercial?

This is the same model used by:

- **MongoDB** — community edition (AGPL→SSPL), commercial edition for SaaS providers
- **Sentry** — open-source (BSL), commercial for hosted Sentry.io
- **Plausible Analytics** — open-source (AGPL), commercial managed hosting
- **Grafana** — open-source (AGPL), commercial Grafana Enterprise

It works because:

1. The open-source license is genuinely strong — anyone can run, modify, and self-host the full thing.
2. The commercial license exists for use cases where AGPL is incompatible with the buyer's business model.
3. Both versions are the same code. There's no "open core" with crippled features. The differentiation is purely legal.
4. Revenue from commercial licenses funds development that benefits the open-source community.

We chose this model because the alternative — pure permissive (MIT/BSD) — would let large vendors fork Pisces Moon OS, integrate it into proprietary products, sell it to your local government, and never contribute anything back. AGPL prevents that. The commercial path lets the rare entity who genuinely can't use AGPL get what they need without breaking the model.
